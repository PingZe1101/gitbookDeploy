typeof、instanceof

  1. typeof
    1. 可以判断是哪种简单数据类型（原始类型）
    2. 可以判断是否为function
    3. 可以判断是否为复杂数据类型（引用类型），判断复杂数据类型时结果均为“object"，不再细分

  2. instanceof

    1. 检查 右侧构造函数的原型对象（prototype） 是否在 左侧对象的原型链（proto）上

    3. 方法右侧 只能是构造函数，将调用其 原型对象prototype上的isPrototypeof()方法，否则将会报错 VM603:1 Uncaught TypeError: Right-hand side of 'instanceof' is not callable ```javascript // 例一 function Person() { this.job = 'teacher'; } const hsy = new Person(); console.log(hsy instanceof Person); // true // hsy instanceof Person <==> Person.prototype.isPrototypeof(hsy);

this指向

  1. long long age的认知
    1. 普通函数的调用，函数内部的this：指向window
    2. 构造函数的调用，函数内部的this：指向实例化以后的对象
    3. 箭头函数的调用，函数内部的this：继承自父级作用域

  2. 正确的认知：

    1. 普通函数的调用，this的指向在函数定义的时候是确定不了的，只有函数执行的时候才能确定this到底指向谁，实际上this最终指向的是那个调用它的上一级的对象，参考：
      1. 如果一个函数中有this，但是它没有被上一级的对象所调用，那么this指向的就是window
      2. 如果一个函数中有this，这个函数有被上一级的对象所调用，那么this指向的就是上一级的对象
      3. 如果一个函数中有this，这个函数中包含多个对象，尽管这个函数是被最外层的对象所调用，this指向的也只是它上一级的对象
    2. 构造函数的调用
      1. 构造函数中没有显式return的情况：函数内部的this：指向实例化以后的对象
      2. 构造函数中有显式return的情况：
        1. 如果return出来的是对象，this指向这个对象
        2. 如果return出来的不是对象，this还是指向实例化后的对象：return null比较特殊，属于这种（也挺容易理解，因为 null本身归类到的是简单数据类型，而且 null.属性 会报错）
          1. 创建对象的两种方式：对象字面量 和 new表达式，使用new操作符的形式创建对象发生了什么
            1. 创建一个空对象
            2. 将空对象的 隐式原型对象proto 指向 构造函数的 显示原型对象prototype
            3. 将构造函数的this指向空对象
            4. 执行构造函数中的代码
            5. 对构造函数return的值做判断，然后返回对应的值
              1. 如果没有显示return 或者 显示return出来的不是对象，则返回前面创建的对象
              2. 如果return出来的是个对象，则返回这个对象
      3. 补充：构造函数的方式 实例化对象，在不需要传实参的情况下可加括号，也可不加括号（推荐➕括号，符合习惯且语意上更好理解）

3. [改变this指向](https://www.runoob.com/w3cnote/js-call-apply-bind.html)，call、apply、bind都是Function.prototype上的方法，三者实现的功能相同，只是入参 和 使用方法上稍有不同，可以相互替换使用
   1. call: fn.call(this, arg1, arg2..., argn)
   2. apply: fn.apply(this, [arg1, arg2..., argn])
   3. bind: fn.bind(this, arg1, arg2..., argn)()，call 和 bind的区别在于后者返回的是一个函数，需要调用后才能执行
      1. 常见使用场景：
         1. 将类（伪）数组转化为数组：Array.prototype.slice.call(类数组) || Array.prototype.slice.apply(类数组) || Array.prototype.slice.bind(类数组)()
         2. 判断数据类型：Object.prototype.toString.call(某类型的数据) || Object.prototype.toString.apply(某类型的数据) || Object.prototype.toString.bind(某类型的数据)()
```javascript
  // 手动实现bind（使用call || apply）
  Function.prototype.myBind = function () {
    // 将类数组转化为数组
    const argumentsArr = Array.prototype.slice.call(arguments);
    // const argumentsArr = Array.from(arguments);
    // 实参中 第一个参数 为this指向
    const targetPoint = argumentsArr.shift();
    // 当前函数内部 this指向实例化以后的对象，缓存一下供return中使用
    const self = this;
    return function () {
      self.call(targetPoint, ...argumentsArr);
      // 或者
      // self.apply(targetPoint, argumentsArr);
    }
  }

  // 手动实现call（使用apply || bind）
  Function.prototype.myCall = function () {
    const argumentsArr = Array.prototype.slice.apply(arguments);
    const targetPoint = argumentsArr.shift();
    this.apply(targetPoint, argumentsArr);
    // this.bind(targetPoint, ...argumentsArr)();
  }

  // 手动实现apply（使用call || bind）
  Function.prototype.myApply = function () {
    const argumentsArr = Array.prototype.slice.call(arguments);
    const targetPoint = argumentsArr.shift();
    // apply 第二个参数为 数组，上行代码已经shift掉了一个最前面的项，所以传入call 直接解构第0项就ok了
    this.call(targetPoint, ...argumentsArr[0]);
    // this.bind(targetPoint, ...argumentsArr[0])();
  }

  // 封装好之后的 myBind、myCall、myApply 就可以像 bind、call、apply 一样使用了，函数内部的this指向调用它的上一级对象

      1. 什么是类（伪）数组：是一种按照索引存储数据并有length属性的对象，本质是对象
      2. 常见的伪数组：jQuery对象、arguments
        1. 在使用 Object.prototype.toString.call()方法处理 jQuery对象 和 arguments时，得出来的结果是 "[object Object]" 和 "[ojbect, Arguments]"，这个不影响它们都是 伪数组 的本质
    2. arguments上的属性：
      1. callee：指向当前执行函数的调用
      2. length：指向传递给当前函数的实参数量
    3. 将类数组转化为数组
      1. Array.prototype.slice.call(类数组)
      2. Array.from(类数组)
      3. [...类数组]

    4. 注意：arguments不可用于箭头函数

      • 一个函数的返回结果只依赖其参数，并且执行过程中没有副作用
        • 只依赖其参数：返回值只和函数参数有关，与外部无关。无论外部发生什么样的变化，函数的返回值都不会改变
        • 执行过程中没有副作用：函数执行的过程中对外部产生了可观察的变化，我们就说函数产生了副作用。例如修改外部的变量、调用DOM API修改页面，发送Ajax请求、调用window.reload刷新浏览器甚至是console.log打印数据，都是副作用

  1. hasOwnProperty

    1. 判断属性（键名）是否为对象的 自定义属性，而非原型对象上的属性

              2. 属性一般在constructor中进行赋值，每个实例化对象一份

    1. match() 方法可在字符串内检索指定的值，或找到一个或多个正则表达式的匹配
    2. match()方法的返回值为：存放匹配结果的数组。
  3. replace方法
    1. replace() 方法用于在字符串中用一些字符替换另一些字符，或替换一个与正则表达式匹配的子串
    2. replace方法的返回值为：一个新的字符串

  4. 注：以上2个方法的参数在使用正则表达式时主要添加全局g，这样才能对字符串进行全部匹配或者替换

    <!DOCTYPE html>
<html lang="zh">
 <head>
     <meta charset="UTF-8" />
     <title>JavaScript中字符串的match与replace方法</title>
 </head>
 <body>
     <script type="text/javascript">
         var str = "1 plus 2 equal 3";
         //match方法返回值为数组
         var arr = str.match(/[0-9]/g)
         console.log(arr); // ["1", "2", "3"]

         var new_str = str.replace(/[0-9]/g, 'newstr');
         //replace方法返回值为新的字符串
         console.log(new_str); // new_str plus new_str equal new_str
     </script>
